01

GTD Mental Model

Understand the logic before touching Todoist. Every decision below flows from this.
Core idea Your brain is for thinking, not storing. Every open loop costs mental energy. GTD externalizes everything into one trusted system so your mind is free to focus.

The 5-Step Flow

1
Capture
Everything that has your attention goes into an inbox. Don't judge or organize it. Just get it out. Todoist inbox is your primary capture point.
2
Clarify
Process each inbox item. Is it actionable? What's the next physical action? If not actionable: trash, defer to Someday, or leave in its reference tool.
3
Organize
Put clarified items in the right place: correct project, right label, correct priority. Waiting-for items get the @waiting-for label and stay in their original project.
4
Reflect
Weekly review. Clear inbox, scan all projects, review waiting-for, check Someday projects. This is what keeps the system alive and trusted.
5
Engage
Do the work. Use filter views to decide what to work on now based on context, energy, and available time.

The 2-Minute Rule

If a task takes under 2 minutes, do it immediately. Don't capture it. The overhead of adding, organizing, and finding it later takes longer than just doing it now.

Tool Roles — Clear Boundary

✅
Todoist
The GTD system: areas, projects, tasks, labels, filters, reminders, someday. Everything actionable lives here.
📝
Obsidian
Project notes and weekly review output. That's it. Not a task manager, not a reference archive.
📅
Calendar
Time-specific commitments: meetings, standups, events with a fixed time. Never put these in Todoist.
🗂️
Reference Tools
Confluence, email archive, Readwise. Non-actionable material. Not in Todoist or Obsidian.
02

Areas & Projects

Two levels. Areas are ongoing responsibilities. Projects are outcomes requiring multiple actions — each one flat inside its Area.
GTD Horizon Model Areas of Responsibility sit above Projects. An Area never completes — it's an ongoing role. A Project completes when its outcome is achieved. "Work" is your Area. "Deliver API v2 by June" is a Project inside it.

Areas vs Projects

🗺️
Area (folder in Todoist)
Ongoing responsibility that never finishes. Groups related projects. During weekly review, ask: "Am I giving this area enough attention?"
🎯
Project (inside an Area)
Any outcome requiring 2+ actions. Has a clear finish line. Flat — no sub-projects. When done, archive it immediately.
💡
Someday (inside each Area)
A holding project for ideas not ready to act on. Review weekly. Not a label — a dedicated project per Area so it doesn't pollute active project views.

Your Todoist Structure

📥Inboxeverything lands here first — process daily
📁WorkArea — Ingenico responsibilities
├──🎯Ingenico Project Aone project per initiative, flat
├──🎯Ingenico Project B
├──🎯Ingenico Project C
├──📋Admin & Opsrecurring reports, team admin
└──💡Someday / Workideas not ready to act on
📁PersonalArea — life outside work
├──👨‍👩‍👧Family & Kids
├──💰Finance
├──🏠Home
├──📚Learningactionable steps only, not article saves
└──💡Someday / Personalideas not ready to act on
📁SystemArea — GTD maintenance
└──🔄Weekly Reviewrecurring checklist lives here
Someday is a project, not a label Tasks in Someday have no @next-action label. They sit quietly in the project until your weekly review. This keeps them completely out of your active task filters.

Rules

1
Each Ingenico project = one Todoist project, flat
No sub-projects. All tasks for that initiative live directly inside the project. If a project gets too large to manage, that's a signal to break it into two separate projects.
2
No "Waiting For" project
Waiting-for items stay in their original project with the @waiting-for label. Use a filter to see all of them across projects at once.
3
Archive completed projects immediately
Done project? Archive it in Todoist right away. Dead projects in the list create noise and slow down your weekly review.
4
Use Areas for the weekly balance check
During weekly review, glance at each Area folder. If Family & Kids has zero active tasks this week, that's worth noticing.
03

Reference

Non-actionable material you might need later. Knowing where this lives prevents you from polluting Todoist with non-tasks.
Key principle Reference is not actionable. It never belongs in Todoist. And it doesn't belong in Obsidian either — that's only for project notes and weekly reviews. Reference lives in your existing work tools.

Where Reference Lives

📘
Confluence
Process docs, architecture decisions, runbooks, project specs. Already your team's source of truth. Don't duplicate in Obsidian.
📧
Email Archive
Past decisions, approvals, client threads. Search when needed. Don't re-file anywhere else.
📖
Readwise Reader
Articles to read and highlights. AI summaries. Removes the "article guilt" backlog. Your Learning project holds actionable steps only — not the articles themselves.
🔖
Browser Bookmarks
Quick-access links and tools. Light folder structure. Don't copy these into Todoist as tasks.

Reference vs Task Decision

?
Is there an action attached?
Yes: it's a task. Capture to Todoist inbox. The reference material stays in its source tool.
?
Is it an article to read?
Save to Readwise Reader. Only add to Todoist if it has a real deadline or a dependency someone is waiting on.
?
Is it a work document?
Lives in Confluence or your shared drive. Don't copy to Obsidian unless you're actively writing or editing it as part of a project.
✓
No action, no obvious home?
Probably not worth keeping. Trash it. GTD reference should only contain things you'll genuinely look up — not everything that feels vaguely useful.
Practical rule If you can find something in 30 seconds by searching your existing tools (email, Confluence, Slack), you don't need to re-file it anywhere. Only create deliberate reference storage for things that are genuinely hard to find when needed.
04

Labels

Labels are GTD statuses and contexts. Keep them minimal. Every label you add is a label you must maintain.
Someday is not a label Someday items live in the Someday project, not labeled in active projects. This keeps them completely out of your next-action filters without any extra effort.

GTD Status Labels (required)

@next-action → what to do next on this project @waiting-for → blocked, someone else's move — stays in original project @agenda → raise next time you meet this specific person

When to Use @next-action

Tasks with a due date don't need this label. Your date-based filters (Today, This Week) already surface them automatically.

Use @next-action only for tasks without a due date that are still the current thing to work on for a project. Without it, those tasks are invisible in your filters.

📅
Has a due date
No label needed. It shows up in Today's Focus and This Week automatically. Examples: reports, client deadlines, recurring tasks.
🎯
No due date
Add @next-action so it appears in your Next Actions filter. Examples: "Review architecture proposal", "Draft project scope" — real work with no hard deadline.
Best practice Each active project should have exactly one @next-action task at any time. When you complete it, assign @next-action to the next task in that project. This keeps every project moving without needing due dates on everything.

Context Labels (optional, add when you need them)

@deep-work → needs uninterrupted focus block @quick → under 15 min, do between meetings @calls → needs a phone or video call
Start with less Use only the 3 GTD status labels for the first few weeks. Add context labels only when you find yourself genuinely needing to filter by them. Most people add too many labels upfront then abandon them.

Examples in Practice

Draft risk assessment for Q3 rollout
@next-action@deep-work
Follow up: John → deployment date (asked May 5)
@waiting-for
Ask Sarah about API rate limit decision
@agenda

Waiting For: Add Context in the Task Name

Include who you're waiting on and when you asked, directly in the task name. Your weekly review Waiting For filter will be much faster to process.

Good: Follow up: John → deployment date confirmation (asked May 5) Follow up: Finance → Q2 budget approval (asked Apr 28) Bad — no context: John budget Waiting for deployment
05

Priority System

4 levels. Use them sparingly or they lose all meaning.
P1 — URGENT

Real deadline today or tomorrow. Serious consequence if missed. Use rarely. If everything is P1, nothing is.

P2 — IMPORTANT

This week, high impact. Your main focus. Most @next-action tasks should sit here.

P3 — NORMAL

Good to do this week but not blocking anything. Do after P1 and P2.

P4 — LOW

Someday, low stakes. Todoist treats unset priority as P4 by default.

Common mistake Overusing P1. If you regularly have 10+ P1 tasks, you have a prioritization problem, not a task management problem. P1 should feel uncommon.

Priority + Label Together

Priority = how urgent. Label = what state the task is in. Both together make your filters useful.

Send incident report to stakeholders by 5pm
@next-actionP1
Review sprint plan for next release
@next-action@deep-workP2
06

Capture

Get everything out of your head fast. Friction at this step kills the whole system.
Golden rule Inbox first, always. Never add a task directly to a project during capture. Dump to inbox, organize later during clarify.

Capture Methods (Pro)

⌨️
Quick Add (Q key)
Desktop shortcut. Type naturally: "Review API docs tomorrow p2" — Todoist parses date and priority automatically.
📱
Mobile Widget
Add Todoist widget to home screen. One tap to capture when away from desk.
🎙️
Ramble (Pro)
Voice capture. Speak naturally, Todoist creates a structured task. Good for commutes or back-to-back meetings.
📧
Email Assist (Pro)
Forward an email to your Todoist address. AI extracts the action item automatically. Useful for email-heavy days.

Natural Language Examples

"Call client about contract tomorrow at 2pm p1" → Due: tomorrow 2pm, Priority: P1 "Send weekly report every friday p2 #Work" → Recurring: every Friday, Priority: P2, Project: Work "Follow up with John every 2 weeks" → Recurring: every 2 weeks

What Not to Do During Capture

Don't add labels. Don't set priority. Don't pick a project. Just type the task name and hit Enter. Organize during clarify. Zero friction at capture is the goal.

07

Clarify & Organize

Process your inbox once or twice a day. Inbox zero is the regular target, not a permanent state.

Decision Tree

?
Is it actionable?
No: trash it, move to a Someday project, or leave it in its reference tool (Confluence, email, Readwise). Yes: continue.
?
Does it take under 2 minutes?
Yes: do it now, delete the inbox item. No: continue.
?
Is it time-specific?
Yes: put it on your calendar, not in Todoist. No: continue.
?
Am I waiting on someone else?
Yes: add @waiting-for, include who + when asked in the task name, move to the relevant project. No: continue.
✓
It's a next action
Add @next-action, set priority, move to the correct project. This is what you'll actually work on.
Don't let inbox pile up Process inbox once in the morning and once after lunch. Skip 3 days and it becomes anxiety instead of relief. The system only works if the inbox stays current and processable.

Filters are where GTD comes alive. These are the views you use daily. Pro unlocks unlimited filters.
How to create filters Todoist sidebar → Filters & Labels → Add filter → paste query → name and save. Pin your 3 most-used to the sidebar.

09

Recurring Tasks

Set once, forget. Todoist regenerates automatically when you complete them.
📅
Calendar (not Todoist)
Daily standup, weekly team sync, monthly 1:1s — anything with a fixed time slot
✅
Todoist Recurring
Status reports, GTD review, expense submissions, finance check — tasks with a cadence but no fixed time
Work vs System: one simple rule If someone at Ingenico cares about the output, it belongs in Work / Admin & Ops. If only you care about it (it keeps your GTD system running), it belongs in System. Worklogs, effort reports, project reports → Work. Inbox processing, weekly review → System.

Syntax Examples

Daily "Process inbox" due: every weekday at 9am Weekly "GTD Weekly Review" due: every friday at 3pm "Send weekly status report" due: every monday at 10am Monthly "Review personal finances" due: every 1st of the month Every N weeks "Follow up with mentor" due: every 2 weeks

10

Weekly Review

The heartbeat of GTD. If you skip this consistently, the system dies. Target: under 20 minutes every Friday.
Why reviews break Most people skip reviews because they take too long. The fix is a tight checklist you actually follow, not a perfect system. 20 minutes max.

Interactive Checklist

Tap to check off. Reset each week.

Clear inbox to zero. Process every item using the clarify decision tree.
~5 min
Review Today filter. Anything overdue? Reschedule or delete. No guilt, just a decision.
~2 min
Review Waiting For filter. Anything stuck too long? Send a follow-up. Mark done if resolved.
~3 min
Scan each Area folder. Are all responsibilities covered? Any area with zero active tasks this week?
~2 min
Check each active project. Does each project have at least one @next-action? If not, add one or it stalls.
~4 min
Open Someday projects. Anything to activate now? Anything to delete? Prune aggressively.
~1 min
Write weekly note in Obsidian. Completed tasks, waiting-for status, next week's focus, one reflection line.
~3 min
11

Obsidian Role

Obsidian has exactly two jobs in this system. Not a task manager. Not a reference archive.
Scope Obsidian = project notes + weekly review output. Everything actionable stays in Todoist. Everything reference stays in your existing work tools.

Job 1: Project Notes

When a Todoist project has context too rich for a task description — background, decisions, meeting notes, links — create one Obsidian note for it. Name it the same as the Todoist project.

Example: Ingenico-Project-A.md ## Goal Deliver API v2 to production by June 30. ## Key Decisions - Chose REST over GraphQL (May 2) - Rate limit: 1000 req/min agreed with client ## Meeting Notes May 5 — Kickoff: agreed on 3-sprint plan... ## Links Jira: Epic XYZ | Confluence: Spec doc

Job 2: Weekly Review Output

After completing your Todoist weekly review checklist, write a short weekly note. This is your done log and next-week anchor.

Example: 2025-W19.md ## Completed - Closed API epic → unblocked team, shipped 3 days late - Reviewed Q2 budget → flagged 12% overspend to finance - Sent compliance docs to legal ## Waiting For - John: deployment window (asked May 6) - Legal: compliance sign-off (sent May 7) ## Next Week Focus - Finalize release plan for v2.1 - Family trip logistics ## Reflection What slowed me down this week?
Keep it manual for now During weekly review, open Todoist completed tasks view, pick what mattered, write 5-8 bullets. Takes 3 minutes. Build the habit for a month before thinking about automating anything.

Quick Reference

Everything condensed. Bookmark this section.

Labels

@next-action → what to do next on this project @waiting-for → blocked, stays in original project (not a separate project) @agenda → raise in next meeting with this person @deep-work → needs uninterrupted focus (optional) @quick → under 15 min (optional) No @someday label — Someday is a project, not a label

Priority

P1 → urgent, real deadline today/tomorrow — use rarely P2 → important this week, main focus P3 → normal, do after P1/P2 P4 → low/someday, no urgency

Clarify Decision Tree

Inbox item → Not actionable? → trash / Someday project / leave in reference tool Under 2 min? → do it now, delete Time-specific? → calendar (not Todoist) Waiting on someone? → @waiting-for + who + date, stays in original project Otherwise → @next-action + priority + move to correct project

Key Filters

Today's Focus → today | overdue Next Actions → @next-action & !@waiting-for Waiting For → @waiting-for Deep Work → @next-action & @deep-work Quick Wins → @next-action & @quick This Week → 7 days & @next-action Someday Review → #Someday / Work, #Someday / Personal

Weekly Review (20 min)

1. Clear inbox to zero (~5 min) 2. Review overdue items (~2 min) 3. Review Waiting For filter (~3 min) 4. Scan each Area folder (~2 min) 5. Check each project has @next-action (~4 min) 6. Open Someday projects, prune (~1 min) 7. Write weekly note in Obsidian (~3 min)

Tool Roles

Todoist → areas, projects, tasks, labels, filters, reminders Calendar → fixed-time events (meetings, standups) Obsidian → project notes + weekly review output only Reference → Confluence, email, Readwise (not Todoist or Obsidian)
Where to start Create Area folders first. Add your current Ingenico projects inside Work, flat. Set up 3 essential filters (Today's Focus, Next Actions, Waiting For). Do your first weekly review. One step at a time.
